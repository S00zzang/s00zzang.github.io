    CINEMATE

    2024-11-14 ~ 2025-01-06
    November 14, 2024

    SUMMARY#

    A platform where users can analyze their movie tastes to find people with similar preferences and chat in real time.

    GOAL#

    Based on each user’s top 10 movie list, personal preferences are analyzed using similarity algorithms. The system provides a list of users with similar tastes and enables real-time chat.

    PARTICIPANT#

    5 (AA)

    ROLE#

    Frontend

    TOOLS#

    • Frontend: Flutter
    • Backend: FastAPI, WebSocket, Kafka
    • Data Analysis: Apache Spark
    • Deployment & Management: AWS EC2, Docker, Airflow
    • DB: Mongo DB

    ARCHITECTURE#

    Project architecture DB architecture

    POINT#

    • Data pipeline that processes user movie data, calculates similarity, and stores results in MongoDB Airflow - similarity.py

      • Connect to MongoDB, verify whether a collection corresponding to the current run date already exists, and delete it if present
      • Ensure data integrity by checking for empty similarity results
      • Automatically store similarity data in a time-based collection for future access

    • Core real-time messaging system using WebSocket Chat server - main.py

      • Import previous conversations from ‘get_previous_messages(KAFKA_TOPIC)’ and deliver to clients via WebSocket
      • Messages processed through Kafka are saved to MongoDB using ‘save_message_to_mongo’
      • Messages processed through Kafka are saved to MongoDB using ‘asyncio.gather’

    TROUBLE SHOOTING#

    • The API has certain limitations that restrict its usage
      • The original plan was to analyze music preferences using the Spotify API. However, due to an announcement that Spotify no longer provides the music analysis API, the plan was switched to use the TMDb API.
      • To avoid over-reliance on a single API, alternative movie information APIs were prepared in advance.
      • Although more tedious, results were stored in the database from the testing phase to ensure that any issues with API calls would not disrupt service.
